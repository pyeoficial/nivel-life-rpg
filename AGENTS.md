# NIVEL — reglas de implementación

- Producto mobile-first, interfaz y mensajes en español.
- Mantener el tono amable: progreso y autonomía, nunca culpa o castigos destructivos.
- D1 es la fuente de verdad; el cliente no decide XP, monedas ni canjes.
- Toda operación económica debe ser calculada y validada en el servidor.
- No guardar secretos ni datos sensibles en el repositorio.
- Antes de cerrar una iteración: ejecutar lint, migraciones, build y prueba del flujo principal.
- Preservar PWA, accesibilidad, áreas táctiles y `prefers-reduced-motion`.
- El MVP debe poder demostrarse sin depender de una API de IA pagada.

## Definition of done

Una función está terminada cuando funciona en móvil, persiste tras recargar, maneja errores, se ve integrada al sistema visual y no rompe el build.
